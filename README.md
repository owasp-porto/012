# OWASP Porto #12 – Cerberus — Presentation

A self-contained, offline [Reveal.js](https://revealjs.com/) slide deck for the
OWASP Porto Chapter meetup **#12 – Cerberus** (July 15th, 2026 · 42 Porto).

18 slides, fixed **1440×900** canvas, OWASP-branded (deep navy + white, orange &
azure accents). Everything it needs (fonts, JS, images, QR codes) is bundled — no
internet connection required to present.

---

## Running the deck

The deck **must be served over HTTP** — opening `index.html` directly via
`file://` breaks font and speaker-notes loading (they are `fetch`ed).

### Option A — npm (http-server)

```bash
npm install        # installs http-server (first time only)
npm start          # runs: http-server -p 80 .
```
Then open **http://localhost/**

> Port 80 is privileged on macOS/Linux, so `npm start` may need `sudo`.
> To use a non-privileged port instead, run http-server directly:
> ```bash
> npx http-server -p 8000 .     # then open http://localhost:8000/
> ```

### Option B — Python (no install)

```bash
python3 -m http.server 8000     # then open http://localhost:8000/
```

### Option C — any static server

Anything that serves the folder over HTTP works, e.g. `npx serve .`.

Present in **fullscreen** (press `F`) for the best result.

## Keyboard shortcuts

| Key(s) | Action |
|--------|--------|
| `→` / `Space` | Next slide (or fragment) |
| `←` | Previous slide |
| `F` | Toggle fullscreen |
| `S` | Open speaker-notes view (shows the notes for each slide) |
| `O` / `Esc` | Slide overview (grid of all slides) |
| `B` | Black out the screen (pause) |
| `M` | Mute / unmute audio |
| `+` / `−` | Volume up / down |
| `?` | Show the keyboard-shortcuts panel (top-right `?` icon) |
| `Home` / `End` | Jump to first / last slide |

The deck is presented **silently by default** (no music, no sound effects), so the
audio keys usually do nothing unless audio is enabled in `config.js`.

## Navigation & the progress bar

- The **sectioned bar at the bottom** shows where you are. The deck has 7
  sections: **Welcome · OWASP · Porto Chapter · Next · News · Today's Event ·
  Close**. The active section is highlighted; the orange tick marks the current
  slide within it; the slide number (e.g. `11 / 18`) sits centered just below.
- **Hover** any point on the bar to see a tooltip with that slide's title;
  **click** to jump straight to it.

## URL options

Append these to the URL to override defaults (handy for rehearsing):

| Parameter | Effect |
|-----------|--------|
| `?fragments=1` | Reveal bullet/card content step-by-step instead of all at once |
| `?audio=0` | Disable all audio |
| `?music=0` / `?sfx=0` | Disable music / sound effects only |
| `?volume=0.25` | Set master volume (0–1) |

You can also deep-link to a slide with the hash, e.g. `…/index.html#/10`.

## Editing the deck

| File | What it controls |
|------|------------------|
| `index.html` | The slides themselves (markup) |
| `config.js` | Options: fragments on/off, progress-bar mode, audio/music/sfx, volume |
| `notes.json` | Speaker notes + per-slide titles (the bar's hover tooltips) |
| `theme/owaspporto.css` | Brand theme (colours, layout, progress bar) |
| `assets/` | Images: cover, sponsor logos, people/keynote photos, **QR codes** |
| `assets/qr-feedback.png` | Feedback-form QR (regenerate if the form link changes) |

**Regenerating the feedback QR** (if the form URL changes):
```bash
python3 -m venv .venv && .venv/bin/pip install "qrcode[pil]"
.venv/bin/python -c "import qrcode; qrcode.make('https://forms.gle/7SV6VzUhoMHcCKVE9').save('assets/qr-feedback.png')"
```

## Exporting to PDF

The native Reveal `?print-pdf` route is unreliable for this theme. Export by
screenshotting each slide at 2× in a headless browser and combining the images,
e.g.:

```bash
# serve the deck (see above), then for each slide #/0 … #/17:
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless=new \
  --disable-gpu --hide-scrollbars --window-size=1440,900 --virtual-time-budget=6000 \
  --screenshot=slide-00.png "http://localhost:8000/index.html?audio=0#/0"
# …then assemble slide-*.png into a PDF (e.g. with reportlab or ImageMagick).
```
